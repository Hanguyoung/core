The properties of the Transmogrifier class are listed below. For a complete list of Transmogrifier class members, see the Transmogrifier Members topic.

Public Instance Properties

AlignmentType Set the bit alignment style for the encoded EXI stream.
BlockSize Set the size, in number of values, of the information that will be processed as a chunk of the entire XML stream. Reducing the block size can improve performance for devices with limited dynamic memory. Default is 1,000,000 items (not 1MB, but 1,000,000 complete Attribute and Element values). Block size is only used when the EXI stream is encoded with EXI-compression.

GrammarCache Set the GrammarCache used in transmogrifying XML data to EXI.
OutputCookie Tells the encoder whether to or not to start the stream by adding an EXI cookie.
OutputOptions Set the header output options. Choices are set using the enumeration. Options are all, lessSchemaID (that is, all values except for the SchemaId), or none.
OutputStream Set an output stream to which encoded streams are written.
PrefixUriBindings @y.exclude
PreserveLexicalValues Set to true to preserve the original string values from the XML stream. For example, a date string might be converted to a different format when interpreted by the Transmogrifier. Preserving the lexical values ensures that the identical strings are restored, and not just their logical values.
PreserveWhitespaces Set to true to preserve whitespace (for example, spaces, tabs, and line breaks) in the encoded EXI stream. By default, non-essential whitespace is removed from the encoded stream.
SAXTransmogrifier Returns the SAXTransmogrifier, which implements both the ContentHandler and LexicalHandler. SAX programmers can connect the SAXTransmogrifier to their favorite XML Parser to convert SAX events into an EXI stream.
ValueMaxLength Set the maximum length of a string that will be stored for reuse in the String Table. By default, there is no maximum length. However, in data sets that have long, unique strings of information, you can improve performance by limiting the size to the length of strings that are more likely to appear more than once.
ValuePartitionCapacity Set the maximum number of values in the String Table. By default, there is no limit. If the target device has limited dynamic memory, limiting the number of entries in the String Table can improve performance and reduce the likelihood that you will exceed memory capacity.
